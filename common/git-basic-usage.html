      # Gitの基本の流れ

      ここでは、Gitを利用したバージョン管理の基本的な流れについてまとめます。

      • 全体像
        • 1. プロジェクトの作成
        • 2. 履歴を残す
        • 3. 反映
      • プロジェクトの作成
        • 新規でプロジェクトを作成する
        • 既存のプロジェクトを持ってくる

      # 全体像

      Gitを利用したバージョン管理のおおまかな流れとしては、下記の通りです。

      # 1. プロジェクトの作成

      バージョン管理をするためのプロジェクト（ソース）をローカル環境に作成します。

      作成方法としては２つあります。

      • 新規で作成する
      • 既存のプロジェクトをローカル環境に持ってくる

      主に使うコマンド

      • git init
      • git clone

      # 2. 履歴を残す

      プロジェクトを作成したら、いよいよ開発です。

      新しくコードを書く、もしくは既存のコードを修正します。

      それが終わったら、Gitに作業したことを伝えてあげる必要があります。

      主に使うコマンド

      • git add
      • git commit

      # 3. 反映

      今までの作業は全てローカル環境での作業になります。

      このままだと、他のメンバーはあなたの書いたコードを利用することができません。

      なので、他のメンバーも利用できるようにリモート環境に反映させてあげる必要があります。

      主に使うコマンド

      • git push

      # 既存のプロジェクトを持ってくる

      既にGithub等のソースコードのホスティングサービスで公開されているプロジェクト（以下、リモートリポジトリ）をローカル環境に持ってくる方法についてです。

      git cloneというコマンドを利用することで実現できます。

      基本の形

      git clone [リモートリポジトリのURL]

      # 本プロジェクト（step-up-camp-doc）を持ってくるためのサンプル
$ git clone https://github.com/ytskmt14/step-up-camp-doc.git
# 上記のようにすると、カレントディレクトリにstep-up-camp-docというディレクトリが作成されます。
$ ls
step-up-camp-doc

      注意！

      コマンドの最初についている $ はプロンプトを表しているので、入力は不要です。
